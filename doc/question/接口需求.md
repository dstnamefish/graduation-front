# 认证与用户管理接口需求

## 1. 基础信息

- **认证模块基础路径**：`/api/auth`
- **用户模块基础路径**：`/api/user`
- **内容类型**：`application/json`

## 2. 认证（Auth）模块 API 接口

### 2.1 登录接口

- **方法**：`POST`
- **路径**：`/api/auth/login`
- **请求参数**：
  ```json
  {
    "username": "admin",
    "password": "123456",
    "captchaKey": "captcha_123",
    "captchaCode": "1234"
  }
  ```

  - `username`：用户名（必填）
  - `password`：密码（必填）
  - `captchaKey`：验证码 key（可选）
  - `captchaCode`：验证码（可选）
- **返回数据**：
  ```json
  {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 3600
  }
  ```

  - `accessToken`：访问令牌（JWT）
  - `refreshToken`：刷新令牌
  - `expiresIn`：令牌过期时间（秒）

### 2.2 退出登录接口

- **方法**：`POST`
- **路径**：`/api/auth/logout`
- **请求参数**：无
- **返回数据**：
  ```json
  {
    "success": true,
    "message": "退出成功"
  }
  ```

### 2.3 注册接口

- **方法**：`POST`
- **路径**：`/api/auth/register`
- **请求参数**：
  ```json
  {
    "username": "newuser",
    "password": "123456",
    "email": "user@example.com",
    "phone": "13800138000"
  }
  ```

  - `username`：用户名（必填）
  - `password`：密码（必填）
  - `email`：邮箱（必填）
  - `phone`：手机号（必填）
- **返回数据**：
  ```json
  {
    "id": 1,
    "username": "newuser",
    "email": "user@example.com"
  }
  ```

  - `id`：用户 ID
  - `username`：用户名
  - `email`：邮箱

### 2.4 获取当前用户信息

- **方法**：`GET`
- **路径**：`/api/auth/profile`
- **请求参数**：无（通过 token 认证）
- **返回数据**：
  ```json
  {
    "id": 1,
    "username": "admin",
    "role": {
      "code": "ADMIN",
      "name": "管理员"
    },
    "realName": "张三",
    "avatar": "https://example.com/avatar.jpg"
  }
  ```

  - `id`：用户 ID
  - `username`：用户名
  - `role`：角色信息
    - `code`：角色代码
    - `name`：角色名称
  - `realName`：真实姓名
  - `avatar`：头像 URL

### 2.5 刷新令牌

- **方法**：`POST`
- **路径**：`/api/auth/refresh-token`
- **请求参数**：
  ```json
  {
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
  ```
- **返回数据**：
  ```json
  {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 3600
  }
  ```

## 3. 用户（User）模块 API 接口

### 3.1 获取用户信息

- **方法**：`GET`
- **路径**：`/api/user/info`
- **请求参数**：无（通过 token 认证）
- **返回数据**：

  ```json
  {
    "id": 1,

    "role": {
      "code": "ADMIN",
      "name": "管理员"
    },
    "realName": "张三",
    "avatar": "https://example.com/avatar.jpg"
  }
  ```

## 4. 数据模型设计

### 4.1 用户表（users）

| 字段名     | 数据类型     | 描述                   |
| ---------- | ------------ | ---------------------- |
| id         | INT          | 用户 ID（主键）        |
| username   | VARCHAR(50)  | 用户名（唯一）         |
| password   | VARCHAR(100) | 密码（加密存储）       |
| email      | VARCHAR(100) | 邮箱（唯一）           |
| phone      | VARCHAR(20)  | 手机号                 |
| role_code  | VARCHAR(20)  | 角色代码               |
| role_name  | VARCHAR(20)  | 角色名称               |
| real_name  | VARCHAR(50)  | 真实姓名               |
| avatar     | VARCHAR(255) | 头像 URL               |
| status     | TINYINT      | 状态（1-启用，0-禁用） |
| created_at | DATETIME     | 创建时间               |
| updated_at | DATETIME     | 更新时间               |

### 4.2 令牌表（tokens）

| 字段名        | 数据类型     | 描述            |
| ------------- | ------------ | --------------- |
| id            | INT          | 令牌 ID（主键） |
| user_id       | INT          | 用户 ID（外键） |
| access_token  | VARCHAR(500) | 访问令牌        |
| refresh_token | VARCHAR(500) | 刷新令牌        |
| expires_at    | DATETIME     | 过期时间        |
| created_at    | DATETIME     | 创建时间        |

## 5. 安全考虑

1. **密码加密**：使用 bcrypt 等算法加密存储密码
2. **JWT 配置**：设置合理的过期时间，使用安全的签名算法
3. **验证码**：登录时可选择开启验证码，防止暴力破解
4. **令牌管理**：实现令牌黑名单机制，处理已注销的令牌
5. **输入验证**：对所有输入参数进行严格验证，防止注入攻击
6. **HTTPS**：生产环境使用 HTTPS 协议
7. **CORS 配置**：正确配置跨域资源共享
8. **权限控制**：基于角色的访问控制，确保用户只能访问授权的资源

## 6. 功能需求

1. **用户认证**：登录、退出、注册
2. **令牌管理**：生成、验证、刷新令牌
3. **用户信息**：获取当前用户信息
4. **权限控制**：基于角色的访问控制
5. **用户管理**：用户信息的增删改查（可扩展）
